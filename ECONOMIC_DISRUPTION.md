# Economic Disruption — Concept & Methodology

**For:** anyone reading the `/economy` page, contributors to the codebase, and analysts auditing the methodology.

## What it is, in one sentence

For every news thread Global Perspectives is already tracking, the **Economic Disruption** layer answers: *which financial instruments is this story repricing, in what direction, with what severity, why, and what historical event is the closest analog?*

## The problem it solves

A serious geopolitics reader looks at the day's news and asks two questions the news itself doesn't answer:

1. **Which markets care about this?** (Oil? Gold? EM FX? Equities? Sovereign debt?)
2. **What did it do last time?** (Is this Iraq 2003, COVID 2020, or Brexit 2016?)

Bloomberg Terminal answers #1 for institutional traders at $25K/year. BlackRock's Geopolitical Risk Dashboard answers it for asset managers. Eurasia Group answers it for boards. Nothing public + free does it for the analyst-grade reader who isn't a fund manager.

Economic Disruption is our attempt at that — built on the news pipeline we already run.

## What it shows you

Per news thread, on `/weekly/thread/<id>` (Economy tab) and `/economy`:

- **Severity** band (minor / moderate / severe) + 0-100 score for sorting
- **Confidence** (low / medium / high) + **Horizon** (immediate / days / weeks / months)
- **Instruments** affected (oil benchmarks, equity indices, bond yields, FX, sector ETFs, crypto when relevant) — each with direction (↑/↓/mixed) and magnitude (small/moderate/large)
- **Mechanism** — 1-2 paragraph causal chain explaining *why* the instrument moves, citing specific articles
- **Winners / Losers** — countries, sectors, or companies who benefit or get hurt
- **Historical analog** — the closest reference event from our curated catalog (Aramco 2019, Russia 2022, Brexit 2016, etc.) and what *actually* happened to relevant assets back then
- **Watch signals** — concrete observables to track (tanker AIS data, OPEC emergency calls, etc.)

Cross-cutting views:

- `/economy` — flagship index, severity-grouped, with facet filters (severity / horizon / instrument / country) and "Today's Top Movers" panel
- `/map` — 4th lens "Economy" shows severity-colored rings on affected countries
- `/daily` — "Today's Economic Footprint" section with top 5 instruments + lead disruption
- Cards across `/weekly`, `/weekly/countries`, Home — show inline severity badges so you can spot economy-relevant stories at a glance

## The methodology — what's generated by AI vs computed

This is the load-bearing distinction. **Compute the numbers. Generate the narrative.**

| Layer | Source | Why |
|---|---|---|
| Price levels (BRENT $82.40, etc.) | `MARKETS_DDB_TABLE` snapshot at generation time | LLMs hallucinate point estimates (FAITH benchmark: 10-20% error). We never let them. |
| Direction (↑/↓/mixed) | LLM | Direction is well-documented in literature; LLMs do this credibly |
| Severity / magnitude | LLM, **as enum** (small/moderate/large; minor/moderate/severe) | Never as a %. Forces qualitative honesty. |
| Mechanism narrative | LLM | Synthesizing causal chains from real text is what LLMs are actually good at |
| Historical analog | LLM picks from our curated catalog | LLM picks; we display the **realized historical moves** from the catalog, not a fresh prediction |
| Citations | LLM, validated server-side | Every claim must cite a topicId from the actual thread. Uncited claims are dropped. |
| `asOf` timestamps | System | Honesty contract — every record carries when it was generated and when prices were snapped |

### The four anti-hallucination guards

1. **Closed instrument allowlist.** The LLM may only reference instruments from a fixed list of ~55 tickers (commodities, sovereign yields, major equity indices + ETFs, FX pairs from our Frankfurter snapshot, and BTC/ETH). Any ticker outside the list — including individual stocks like AAPL/TSM/RTX — is dropped server-side. The LLM literally cannot publish a recommendation on something we don't track.

2. **Citation requirement.** Every instrument claim must cite topicIds from the thread's actual articles. The mechanism paragraph must cite topicIds inline (`[topic-abc]`). Claims without citations are dropped. Citations to topicIds outside the thread are dropped.

3. **Tombstones for "no impact."** When a thread has no genuine economic dimension (e.g., a profile of a political figure), the LLM is instructed to return `{"hasImpact": false}` and a tombstone is written. **Refusing to generate is better than confabulating.** This is the discipline that separates analyst tools from content farms.

4. **Magnitude as enum.** Never a percentage. Magnitudes are `small | moderate | large`. Severity is `minor | moderate | severe`. The LLM doesn't get to claim "Brent will rise 4.2%" — only "moderate up move." This is exactly what BlackRock and Eurasia publish, for the same reason: false precision is more harmful than honest banding.

## Historical analog catalog

We maintain a curated table of ~22 reference events (in `amplify/backend/function/newsEconomicImpact/src/economic_analogs.json`) covering:

- Oil/Middle East: Aramco 2019, Russia-Ukraine 2022, Iran sanctions 2018, Houthi Red Sea 2024, Israel-Hezbollah 2006, Israel-Gaza 2023, Qatar blockade 2017
- Currency/EM crisis: Argentina IMF 2018, Turkey 2018-2021, Brexit 2016, SNB unpeg 2015, UK mini-budget 2022
- Trade wars: US-China tariffs 2018-19, Taiwan Pelosi 2022
- Banking/credit: Lehman 2008, SVB collapse 2023, Draghi 2012
- Other: COVID 2020, HK protests 2019, Saudi-Iran restoration 2023, Sahel coups 2020-2023

Each entry records what **actually happened** to specific assets after the event — not a prediction. When the LLM picks an analog (e.g., "this looks like Aramco 2019"), the UI shows the historical moves from 2019, not a fresh forecast for today.

The catalog is intentionally finite. When the LLM cites an event we don't have, we add it (this happened during testing — "Israel-Hezbollah 2006" wasn't in the catalog, so we added it).

## What it deliberately is *not*

- **Not investment advice.** Disclaimed in the UI, in `/disclosures`, and in the underlying schema. We pick directions and severity bands; we don't tell you to buy, sell, or hold anything.
- **Not real-time alerting.** Records refresh roughly daily (07:30 UTC scheduled). For 30-second tape moves, use Bloomberg/Reuters.
- **Not a Bloomberg replacement.** Bloomberg has 29 risk categories cascading to 7M companies. We have ~55 instruments and run on a $0.30/month budget. We work where the analyst question is *"which markets should I be watching today?"*, not *"what's the trade in COTY 9% senior unsecured?"*
- **Not point-precision.** No price forecasts. No "+4.2%". Severity bands only.
- **Not opinionated on the politics.** The methodology cares whether oil moves; it doesn't care whether the war is justified.

## Limitations (be candid)

- **Spurious attribution.** Most daily market moves are driven by Fed/positioning/liquidity, not by the geopolitical headline. We frame things as "consistent with" not "caused by," but the framing is a soft guardrail; readers should still treat us as one input.
- **Selection bias in analogs.** The Aramco 2019 attack is in the catalog because oil spiked. The dozens of Yemen drone strikes that did nothing are forgotten and not in the catalog. We try to include non-reactive precedents too, but the bias is real.
- **LLM taste, not calibration.** Confidence/severity scores are LLM judgment, not calibrated probabilities. A "severe 85" today and a "severe 85" next month aren't necessarily commensurate — yet. (This is what the calibration tracker in [§ Future work](#future-work) is supposed to address.)
- **Coverage gaps.** Our instrument allowlist is conservative on purpose. We don't track Singapore SGX, Saudi TASI, individual sovereign CDS, options skew, commodity-specific futures curves, or anything that requires paid feeds. If a story moves Indonesian rupiah, we'll see USD/IDR via Frankfurter; if it moves Indonesian rupiah forwards 3-month, we won't.
- **Latency.** Daily schedule. A breaking story at 07:31 UTC won't appear in the disruption layer for ~24 hours.

## What's running today (verified live 2026-05-19)

**Backend (AWS Lambda, ap-northeast-1):**
- `newsMarketsData` — ingests FX (Frankfurter, ECB) + bond yields (FRED) + commodities (Stooq) + equity indices + sector ETFs (Stooq) + crypto (CoinGecko). All free data sources, no paid feeds.
- `newsEconomicImpact` — daily 07:30 UTC. Reads top 15 economically-relevant threads. Calls DeepSeek with the closed allowlist + market snapshot + relevant analogs. Validates JSON. Writes to `SummarizeAndPredict` table with `ECON#THREAD#{id}` keys, 21-day TTL.
- `newsSensitiveData` — exposes 3 REST actions: `economic_impact`, `economic_impact_list`, `economic_top_movers`.

**Frontend (React + Vite, served from `docs/` on GitHub Pages):**
- New flagship `/economy` page (3-column EditorialShell with facet filters + severity-grouped list + top movers panel)
- New "Economy" tab on every `/weekly/thread/:id`
- New "Economic Disruption" section on every `/weekly/country/:name` (above Macro Baseline)
- New 4th "Economy" lens on `/map`
- New "Today's Economic Footprint" section on `/daily`
- Severity badges on Home / WeeklyPage / CountryListPage cards

**Marginal cost:** ~$0.30/month at current daily volume.

## How to read a disruption record (the UX vocabulary)

When you open the Economy tab on a thread, here's what to look for:

| Field | What it tells you |
|---|---|
| **Severity (color)** | Red (severe) = headline-driving market event. Amber (moderate) = real but contained. Green (minor) = de-escalation or low-stakes. |
| **Score (0-100)** | Granular sort key. Within "moderate," 65 beats 45 for prominence. |
| **Confidence** | "High" = clear mechanism + multiple cited articles + well-traveled analog. "Low" = directional bet with limited evidence. Hedge accordingly. |
| **Horizon** | When the effect plays out. "Immediate" = today's tape. "Months" = structural repricing. |
| **Instruments** | The ticker + arrow + magnitude. Ignore the magnitude bar's exact length — the band (small/moderate/large) is the signal. |
| **Mechanism** | The "why." If this doesn't make sense to you, distrust the rest of the record. |
| **Winners / Losers** | Country-level winners link to `/weekly/country/:name`. Sectors stay as labels — we don't have sector pages yet. |
| **Historical analog** | The italic line at the bottom. It's pointing to a real past event with a real outcome. The "caveat" line is important — it's how the LLM is saying "but this isn't exactly the same as that." |

## Future work (deferred)

Listed in the implementation plan (`ECONOMIC_DISRUPTION_PLAN.md`) but not yet built:

- **Calibration tracker.** Record every direction call's timestamp + instrument. After 30/60/90 days, compute hit rate vs realized moves from `MARKETS_DDB_TABLE` HISTORY. Publish the hit rate on `/economy`. This is the strongest credibility moat — no LLM-driven public product does this — and we can't build it until we have a few months of history.
- **GPR-style index.** Caldara & Iacoviello (2022) build a Geopolitical Risk Index by counting categorized keyword hits in major newspapers, normalized as a z-score. We could port their method to our 26 RSS feeds. Transparent, peer-reviewed, free to read the math.
- **Country aggregator.** Currently country pages show *thread-level* disruptions filtered by country. A daily LLM pass could synthesize a single coherent country-level disruption record across all of a country's threads.
- **Map transmission arcs.** Producer → importer arcs on the Economy map lens (e.g., Saudi Arabia → Japan when oil spikes). Currently deferred — the ring overlay is the conservative version.
- **Reverse Lambda.** Given a big market move (e.g., DXY breaks 105), find news threads that explain it. Currently we go from news → market direction; the reverse is also useful.

## References (for skeptics)

These are the public sources we lean on, in case you want to audit:

- [Caldara & Iacoviello (2022) Geopolitical Risk Index](https://www.matteoiacoviello.com/gpr.htm) — academic standard. Free data. Peer-reviewed methodology.
- [BlackRock Geopolitical Risk Dashboard](https://www.blackrock.com/corporate/insights/blackrock-investment-institute/interactive-charts/geopolitical-risk-dashboard) — closest public-facing UI precedent. Market-Driven Scenario layer is the conceptual ancestor of our "instruments + mechanism."
- [Kilian (2009) on oil-price geopolitical risk](https://www.eia.gov/finance/markets/reports_presentations/2025/Kilian_EIAWS_2025.pdf) and [Dallas Fed WP 2609 (2026)](https://www.dallasfed.org/~/media/documents/research/papers/2026/wp2609.pdf) — the literature on how oil reacts to Middle East geopolitics is well-developed and we lean on it for stylized facts.
- [FAITH (2025): hallucination benchmark for financial reasoning](https://arxiv.org/abs/2508.05201) — why we never let the LLM emit price-level numbers.
- [FTC Disclosures 101](https://www.ftc.gov/business-guidance/resources/disclosures-101-social-media-influencers) — the legal framing for "not investment advice."

## Testing

Three layers, all passing as of 2026-05-20:

### Backend — validator unit tests (no AWS, no LLM)

31 tests covering the closed instrument allowlist, citation requirement, enum normalization, and tombstone logic. No AWS credentials or DeepSeek calls — runs entirely against the Lambda source with a stubbed module loader.

```bash
node amplify/backend/function/newsEconomicImpact/test/validator.test.js
```

What gets verified:
- Valid input passes through with all fields preserved
- Unknown tickers (AAPL, RTX, BANANA) get dropped server-side
- Instruments with empty `citedTopicIds` get dropped
- Citations to topicIds outside the thread's entries get dropped
- Invalid direction enum (`higher`) → instrument dropped
- Invalid magnitude (`mega-large`) → normalized to `moderate`
- When all instruments are invalid → tombstone (`hasImpact: false`)
- When `citedTopicIds` is empty → tombstone
- FX pairs from the runtime FX allowlist accepted (USD/EUR)
- Allowlist sanity — BRENT/INDA/EIS/BTC/ETH IN, NSEI/TA125/AAPL/RTX/TSM OUT

### Frontend — vitest hook tests

12 tests for `useEconomicImpact` and `useDisruptionsList` covering fetch lifecycle, localStorage cache, null inputs, error fallbacks, and filter passing. Mirrors the existing `useSystemsAnalysis.test.js` pattern.

```bash
cd global-perspectives-starter/frontend
npm test                                               # full suite (149 tests)
npx vitest run src/test/useEconomicImpact.test.js      # just this file
```

### What's deliberately NOT tested in code

- **Real LLM output quality.** Reviewed manually against the red-flag table (random tickers? uncited claims? wrong direction? made-up analogs?). Re-review after material prompt or model changes.
- **End-to-end with real news.** Validated once by running the Lambda against production DDB during deploy. Re-validate after any change to the data pipeline.
- **Click-through UX flows.** Manual `npm run dev` + browser. Per [`feedback_test_ui_in_browser.md`](../../../../.claude/projects/-Users-benlai-Downloads-globalPerspective-v1/memory/feedback_test_ui_in_browser.md): every control gets clicked before claiming done.

## For developers

If you want to extend this:
- **Add an instrument** → edit `INSTRUMENT_ALLOWLIST` in `newsEconomicImpact/src/index.js` AND `STOOQ_INDICES`/`STOOQ_ETFS` in `newsMarketsData/src/index.js` AND the prompt's instrument table builder. The allowlist is the single source of truth — if it's not in there, the LLM can't emit it.
- **Add an analog event** → append to `economic_analogs.json`. The Lambda hot-loads it on each cold-start.
- **Tune severity calibration** → adjust the prompt's severity rubric in `buildPrompt()`. Re-test against existing records to see drift.
- **Change cadence** → modify EventBridge rule `TriggerNewsEconomicImpact`. Currently `cron(30 7 * * ? *)`.
- **Pipeline implementation details** → see `ECONOMIC_DISRUPTION_PLAN.md` and `ARCHITECTURE.md`.
